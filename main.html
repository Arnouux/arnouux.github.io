My Projects

Chessboard recognition with convolutional neural network

Creating a full dataset, training a neural network

GitHub of the project The program has 4 steps as we can see on the picture.

  1. Take a photo of the current chessboard. Using the chessboard from the dataset
  2. Resize the photo to a perfect square on the chessboard. Using another project available on github, using heatmap to replace the board correctly.
  3. Using a convolutional neural network to get the class of the piece on every cells. Cut the board in 64 cells thanks to the perfect square. Get the piece on every cells. Allow the user to modify a piece if one is uncorrect (>99% accuracy).
  4. Use an API to ask one of the best AI which move to play. Translate the board from a 2D array to Forsyth–Edwards Notation (FEN) so that Stockfish's api can understand. Receive information and propose to the user the best move.

Technologies: Python (Keras & Tensorflow)

Neural networks and genetic algorithms

GitHub of the project This project combines neural networks (NNs) to genetic algorithms, in a 2D car race environnement. Here, NNs don't learn by backpropagation according to their errors. Instead, a lot of generations of NNs are created and a genetic algorithm makes the best ones pass to the next generation. Hence, step by step, we find good architectures and parameters for the NN. Steps for the genetic algorithm:

  1. Creation of the generation
  2. Selection of the bests, accorgind to a fitness function calculator (here the distance traveled by the car + checkpoints to avoid turning around)
  3. Crossover: swap randomly selected neurons from 2 NNs
  4. Mutation: change some randomly selected parameters in a NN

Technologies: Python (No ML framework, Pygame)

End-to-end encrypted messaging application

Cryptography and java

GitHub of the project The project was about creating an end-to-end encrypted messaging application, meaning the server sending the messages from one user to another has no way to read the content of it. The application uses an hybrid encryption, because decrypting a message with a symmetric key is faster than with an asymmetric key. The clients then have to share a secret key :

  • On registration, clients are creating a public/private key pair (Asymmetric encryption)
  • Client 1 wants to send a message to client 2
  • Server sends to client 1 the public key of client 2
  • Client 1 generates a secret key (Symmetric encryption) and sends it with the message encrypted with the public key of client 2
  • Client 2 uses their private key to decrypt the message using the secret key
  • Of course, at no moment the server can decrypt the messages

Technologies: Java, keystores & certificates, Bouncycastle, multithreading
